### 1. ✅ macOS Compatibility Fix (opencode_service.sh)

**Problem**: `readlink -f` doesn't exist on macOS BSD version

**Location**: `src/commands/scripts/services/opencode_service.sh` (lines 1-20)

**Solution**:
```bash
#!/bin/bash

# Cross-platform script path detection
if [[ "$OSTYPE" == "darwin"* ]]; then
    # macOS: Try greadlink first (GNU version from Homebrew)
    if command -v greadlink >/dev/null 2>&1; then
        SCRIPT_DIR="$(dirname "$(greadlink -f "${BASH_SOURCE[0]}")")"
    else
        # Fallback: Use cd and pwd
        SOURCE="${BASH_SOURCE[0]}"
        while [ -h "$SOURCE" ]; do
            DIR="$(cd -P "$(dirname "$SOURCE")" && pwd)"
            SOURCE="$(readlink "$SOURCE")"
            [[ $SOURCE != /* ]] && SOURCE="$DIR/$SOURCE"
        done
        SCRIPT_DIR="$(cd -P "$(dirname "$SOURCE")" && pwd)"
    fi
else
    # Linux: Use readlink -f
    SCRIPT_DIR="$(dirname "$(readlink -f "${BASH_SOURCE[0]}")")"
fi
```

**Impact**:
- ✅ Works on macOS (both with and without `greadlink`)
- ✅ Works on Linux
- ✅ OpenCode service now functional on macOS

### 2. ✅ Bash/Zsh Compatibility Fix (wyx-cli.sh)

**Problem**: `${(%):-%N}` is zsh-specific, causes "bad substitution" in bash

**Location**: `wyx-cli.sh` (lines 4-14)

**Solution**:
```bash
#!/bin/bash

# Shell-aware script path detection
if [ -n "$ZSH_VERSION" ]; then
    # Zsh
    WYX_DIR="$(cd "$(dirname "${(%):-%N}")" && pwd)"
elif [ -n "$BASH_VERSION" ]; then
    # Bash
    WYX_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
else
    # Fallback
    WYX_DIR="$(cd "$(dirname "$0")" && pwd)"
fi

export WYX_DIR
```

**Impact**:
- ✅ Works in zsh
- ✅ Works in bash
- ✅ Graceful fallback for other shells

### 3. ✅ Interactive Delete Command

**Feature**: Safe, interactive file/folder deletion with macOS Trash support

**Location**: `src/commands/delete.sh` (245 lines)

**Key Features**:
- Arrow key navigation (↑↓)
- Multi-select with spacebar
- Visual selection indicators [✓]
- Directory size and file count display
- Confirmation prompt before deletion
- Moves to macOS Trash (recoverable)
- Supports both `trash` command and `osascript` fallback

**Implementation**:

```bash
#!/bin/bash

# Colors
GREEN=$(tput setaf 2)
ORANGE=$(tput setaf 3)
RED=$(tput setaf 1)
BLUE=$(tput setaf 4)
CYAN=$(tput setaf 6)
RESET=$(tput sgr0)
BOLD=$(tput bold)

# Check if trash command is available
has_trash_cmd() {
    command -v trash >/dev/null 2>&1
}

# Move item to trash using available method
move_to_trash() {
    local item="$1"
    
    if has_trash_cmd; then
        trash "$item" 2>/dev/null
        return $?
    else
        # Fallback to osascript on macOS
        if [[ "$OSTYPE" == "darwin"* ]]; then
            osascript -e "tell application \"Finder\" to delete POSIX file \"$(cd "$(dirname "$item")" && pwd)/$(basename "$item")\"" >/dev/null 2>&1
            return $?
        else
            sys.log.error "No trash command available. Install 'trash-cli' or use 'rm' manually."
            return 1
        fi
    fi
}

# Get directory size and file count
get_dir_info() {
    local dir="$1"
    if [[ -d "$dir" ]]; then
        local file_count=$(find "$dir" -type f 2>/dev/null | wc -l | tr -d ' ')
        local size=$(du -sh "$dir" 2>/dev/null | cut -f1)
        echo "$file_count files ($size)"
    fi
}

# Interactive file selector with arrow keys
interactive_select() {
    local target_dir="${1:-.}"
    
    # Get list of files/folders (excluding . and ..)
    local -a items=()
    while IFS= read -r line; do
        items+=("$line")
    done < <(ls -A "$target_dir" 2>/dev/null | sort)
    
    if [[ ${#items[@]} -eq 0 ]]; then
        sys.log.warn "No items found in directory"
        return 1
    fi
    
    # Initialize selection array (0 = not selected, 1 = selected)
    local -a selected=()
    for ((i=0; i<${#items[@]}; i++)); do
        selected[$i]=0
    done
    
    local current=0
    local key=""
    
    # Hide cursor
    tput civis
    
    # Function to draw menu
    draw_menu() {
        clear
        sys.log.h1 "Select files to delete (↑↓ to navigate, SPACE to select, ENTER to confirm, q to quit)"
        echo ""
        
        local selected_count=0
        for ((i=0; i<${#items[@]}; i++)); do
            local item="${items[$i]}"
            local full_path="$target_dir/$item"
            local marker="[ ]"
            local color="$RESET"
            
            if [[ ${selected[$i]} -eq 1 ]]; then
                marker="[${GREEN}✓${RESET}]"
                selected_count=$((selected_count + 1))
            fi
            
            # Highlight current line
            if [[ $i -eq $current ]]; then
                color="$BOLD$CYAN"
            fi
            
            # Add directory indicator and info
            if [[ -d "$full_path" ]]; then
                local dir_info=$(get_dir_info "$full_path")
                echo -e "${color}${marker} ${item}/ ${BLUE}(${dir_info})${RESET}"
            elif [[ -L "$full_path" ]]; then
                echo -e "${color}${marker} ${item} ${ORANGE}(symlink)${RESET}"
            else
                echo -e "${color}${marker} ${item}${RESET}"
            fi
        done
        
        echo ""
        sys.log.h2 "→ Selected: ${selected_count} item(s)"
    }
    
    # Main loop
    while true; do
        draw_menu
        
        # Read single character
        read -rsn1 key
        
        # Handle escape sequences for arrow keys
        if [[ "$key" == $'\x1b' ]]; then
            read -rsn2 key
            case "$key" in
                '[A') # Up arrow
                    ((current--))
                    [[ $current -lt 0 ]] && current=$((${#items[@]} - 1))
                    ;;
                '[B') # Down arrow
                    ((current++))
                    [[ $current -ge ${#items[@]} ]] && current=0
                    ;;
            esac
        elif [[ "$key" == " " ]]; then
            # Toggle selection
            if [[ ${selected[$current]} -eq 0 ]]; then
                selected[$current]=1
            else
                selected[$current]=0
            fi
        elif [[ "$key" == "" ]]; then
            # Enter key - confirm selection
            break
        elif [[ "$key" == "q" ]] || [[ "$key" == "Q" ]]; then
            # Quit
            tput cnorm
            clear
            sys.log.info "Cancelled"
            return 0
        fi
    done
    
    # Show cursor
    tput cnorm
    clear
    
    # Check if anything is selected
    local selected_items=()
    for ((i=0; i<${#items[@]}; i++)); do
        if [[ ${selected[$i]} -eq 1 ]]; then
            selected_items+=("${items[$i]}")
        fi
    done
    
    if [[ ${#selected_items[@]} -eq 0 ]]; then
        sys.log.info "No items selected"
        return 0
    fi
    
    # Show selected items and confirm
    sys.log.h1 "Selected items to delete:"
    echo ""
    for item in "${selected_items[@]}"; do
        local full_path="$target_dir/$item"
        if [[ -d "$full_path" ]]; then
            local dir_info=$(get_dir_info "$full_path")
            echo "  ${RED}✗${RESET} ${item}/ ${BLUE}(${dir_info})${RESET}"
        else
            echo "  ${RED}✗${RESET} ${item}"
        fi
    done
    echo ""
    
    # Warning for directories
    local has_dirs=false
    for item in "${selected_items[@]}"; do
        if [[ -d "$target_dir/$item" ]]; then
            has_dirs=true
            break
        fi
    done
    
    if [[ "$has_dirs" == true ]]; then
        sys.log.warn "Warning: Some selected items are directories with contents"
        echo ""
    fi
    
    # Final confirmation
    echo -n "${ORANGE}Move these ${#selected_items[@]} item(s) to Trash? [y/N]:${RESET} "
    read -r confirm
    
    if [[ ! "$confirm" =~ ^[Yy]$ ]]; then
        sys.log.info "Cancelled"
        return 0
    fi
    
    echo ""
    
    # Delete items
    local success_count=0
    local fail_count=0
    
    for item in "${selected_items[@]}"; do
        local full_path="$target_dir/$item"
        if move_to_trash "$full_path"; then
            sys.log.info "✓ Moved to Trash: $item"
            ((success_count++))
        else
            sys.log.error "✗ Failed to delete: $item"
            ((fail_count++))
        fi
    done
    
    echo ""
    if [[ $success_count -gt 0 ]]; then
        sys.log.info "Successfully deleted $success_count item(s)"
    fi
    if [[ $fail_count -gt 0 ]]; then
        sys.log.error "Failed to delete $fail_count item(s)"
    fi
}

# Main execution
if wyxd.arggt "1"; then
    # Use provided path
    target_path="$1"
    if [[ ! -d "$target_path" ]]; then
        sys.log.error "Error: '$target_path' is not a valid directory"
        exit 1
    fi
    interactive_select "$target_path"
else
    # Use current directory
    interactive_select "."
fi
```

### 4. ✅ Global Command Access

**Feature**: All WYX-CLI commands available from any directory without prefix

**Implementation**: `generate-shell-functions.sh`

```bash
#!/bin/bash

# WYX CLI - Shell Functions Generator
# This script generates shell functions for all wyx commands
# so they can be used directly without the 'wyx' prefix

WYX_DIR="$(dirname "$0")"
CSV_FILE="$WYX_DIR/src/data/arg_scripts.csv"
OUTPUT_FILE="$WYX_DIR/wyx-functions.sh"

echo "# WYX CLI Auto-generated Functions" > "$OUTPUT_FILE"
echo "# Generated on $(date)" >> "$OUTPUT_FILE"
echo "# DO NOT EDIT THIS FILE MANUALLY" >> "$OUTPUT_FILE"
echo "# Regenerate by running: $WYX_DIR/generate-shell-functions.sh" >> "$OUTPUT_FILE"
echo "" >> "$OUTPUT_FILE"

# Read all public commands from CSV (skip header, only get public commands)
while IFS=',' read -r id path name public args category_id; do
    # Skip header
    if [ "$id" = "id" ]; then
        continue
    fi
    
    # Only process public commands (public=true)
    if [ "$public" = "true" ]; then
        # Skip commands that might conflict with common shell commands
        # We'll handle 'cd' specially since wyx has its own cd
        if [ "$id" = "cd" ]; then
            echo "# Note: 'cd' is handled by wyx cd for directory aliases" >> "$OUTPUT_FILE"
            echo "wyxcd() { wyx cd \"\$@\"; }" >> "$OUTPUT_FILE"
            echo "" >> "$OUTPUT_FILE"
            continue
        fi
        
        # Generate function for each command
        echo "${id}() { wyx ${id} \"\$@\"; }" >> "$OUTPUT_FILE"
    fi
done < "$CSV_FILE"

echo "" >> "$OUTPUT_FILE"
echo "# End of WYX CLI Auto-generated Functions" >> "$OUTPUT_FILE"

echo "✓ Generated shell functions in: $OUTPUT_FILE"
echo ""
echo "To use these functions, add this line to your ~/.zshrc:"
echo "source $OUTPUT_FILE"
```

is_nemo_project() {

    fi

Standalone validation tool (~120 lines):

```bash
#!/bin/bash

# Nemo Project Validator
# Validates branch names and commit messages for Nemo project standards

# Colors
RED=$(tput setaf 1)
GREEN=$(tput setaf 2)
YELLOW=$(tput setaf 3)
RESET=$(tput sgr0)

# Valid configurations
VALID_TYPES="feature|bugfix|hotfix|tech|spike|bug"
VALID_JIRA_PREFIXES="NEMO2|PROFSD|NEMO3A|NEMOME|NEMONE|NEMOAR|NEMOIT|NEMOSME"

# Validate branch name
validate_branch_name() {
    local branch="$1"
    local pattern="^(${VALID_TYPES})/(${VALID_JIRA_PREFIXES})-[0-9]+[_-].+$"
    
    if [[ "$branch" =~ $pattern ]]; then
        echo "${GREEN}✓${RESET} Valid branch name: $branch"
        return 0
    else
        echo "${RED}✗${RESET} Invalid branch name: $branch"
        echo ""
        echo "Expected format: type/JIRA-ID_description"
        echo "Valid types: feature, bugfix, hotfix, tech, spike, bug"
        echo "Valid JIRA prefixes: NEMO2, PROFSD, NEMO3A, NEMOME, NEMONE, NEMOAR, NEMOIT, NEMOSME"
        echo ""
        echo "Examples:"
        echo "  feature/NEMO2-12345_add-authentication"
        echo "  bugfix/PROFSD-789_fix-login-error"
        echo "  tech/NEMOIT-456_upgrade-dependencies"
        return 1
    fi
}

# Validate commit message
validate_commit_message() {
    local message="$1"
    local pattern="^(${VALID_JIRA_PREFIXES})-[0-9]+: .+"
    
    # Check for exception keywords
    if [[ "$message" =~ (hotfix|buildfix) ]]; then
        echo "${YELLOW}⚠${RESET} Exception commit (hotfix/buildfix): allowed"
        return 0
    fi
    
    if [[ "$message" =~ $pattern ]]; then
        echo "${GREEN}✓${RESET} Valid commit message"
        return 0
    else
        echo "${RED}✗${RESET} Invalid commit message: $message"
        echo ""
        echo "Expected format: JIRA-ID: Message"
        echo "Example: NEMO2-12345: Add user authentication"
        echo ""
        echo "Exception: Use 'hotfix' or 'buildfix' keyword to bypass check"
        return 1
    fi
}

# Extract JIRA ID from branch
extract_jira_id() {
    local branch="$1"
    if [[ "$branch" =~ (${VALID_JIRA_PREFIXES})-[0-9]+ ]]; then
        echo "${BASH_REMATCH[0]}"
        return 0
    fi
    return 1
}

# Main command dispatcher
case "$1" in
    branch)
        validate_branch_name "$2"
        ;;
    commit)
        validate_commit_message "$2"
        ;;
    jira-id)
        extract_jira_id "$2"
        ;;
    current-branch)
        current=$(git symbolic-ref --short HEAD 2>/dev/null)
        if [ -n "$current" ]; then
            validate_branch_name "$current"
        else
            echo "${RED}✗${RESET} Not in a git repository"
            exit 1
        fi
        ;;
    *)
        echo "Nemo Project Validator"
        echo ""
        echo "Usage:"
        echo "  $0 branch <branch-name>        # Validate branch name"
        echo "  $0 commit <commit-message>     # Validate commit message"
        echo "  $0 jira-id <branch-name>       # Extract JIRA ID from branch"
        echo "  $0 current-branch              # Validate current git branch"
        echo ""
        echo "Examples:"
        echo "  $0 branch feature/NEMO2-12345_add-auth"
        echo "  $0 commit \"NEMO2-12345: Add authentication\""
        echo "  $0 jira-id feature/NEMO2-12345_add-auth"
        echo "  $0 current-branch"
        exit 1
        ;;
esac

alias | grep wyx

# If missing, add to shell config:
export WYX_DIR="/path/to/WYX-CLI"
alias wyx="source $WYX_DIR/wyx-cli.sh"
source ~/.zshrc
```

#### Issue 2: "command not found: delete" (or other commands)

**Cause**: Shell functions not sourced

**Solution**:
```bash
# Check if functions file exists
ls -la $WYX_DIR/wyx-functions.sh

# Regenerate functions
$WYX_DIR/generate-shell-functions.sh

# Add to shell config
echo "source $WYX_DIR/wyx-functions.sh" >> ~/.zshrc
source ~/.zshrc
```

#### Issue 3: "bad substitution" error on bash

**Cause**: Old version of wyx-cli.sh with zsh-specific syntax

**Solution**:
```bash
# Pull latest version with fix
cd $WYX_DIR
git pull origin main

# Or manually apply fix (see Implementation #2 above)
```

#### Issue 4: "readlink: illegal option -- f" on macOS

**Cause**: macOS BSD readlink doesn't support `-f` flag

**Solution**:
```bash
# Install GNU readlink (greadlink)
brew install coreutils

# Or pull latest opencode_service.sh with fix
cd $WYX_DIR
git pull origin main
```

#### Issue 5: Delete command shows escape codes instead of colors

**Cause**: Terminal doesn't support ANSI colors or tput

**Solution**:
```bash
# Check terminal support
echo $TERM

# Should be: xterm-256color or similar
# If not, set it:
export TERM=xterm-256color
```

#### Issue 6: OpenCode wrapper times out

**Cause**: Loading full zshrc takes too long

**Solution**:
```bash
# Use lite wrapper instead
$WYX_DIR/opencode-wyx-lite.sh genpass 20

# Or optimize ~/.zshrc to load faster
```

#### Issue 7: "No trash command available"

**Cause**: `trash` command not installed

**Solution**:
```bash
# Install trash command
brew install trash

# Or use osascript fallback (automatic on macOS)
```

#### Issue 8: Git commands fail with "not a git repository"

**Cause**: Running git command in non-git directory (expected behavior)

**Solution**:
```bash
# Navigate to a git repository
cd /path/to/git/repo

# Or initialize new repo
git init
```

#### Issue 9: "Direct push to 'main' is not allowed" (Nemo integration)

**Cause**: Protection against pushing to main branch

**Solution**:
```bash
# Create feature branch
nb feature/JIRA-ID_description

# Make changes and push
push

# Create PR on GitHub
```

#### Issue 10: Functions work in terminal but not in scripts

**Cause**: Non-interactive shells don't load ~/.zshrc

**Solution**:
```bash
# In scripts, source the functions explicitly
source /path/to/WYX-CLI/wyx-functions.sh

# Or use wyx prefix
wyx delete
```

### Git Hooks Integration

**Note**: WYX-CLI works alongside existing git hooks, not replacing them.

**Nemo Project Hooks** (example):
```bash
# .git/hooks/commit-msg
# Validates JIRA ID in commit message
# WYX-CLI complements this by auto-prefixing JIRA IDs

# .git/hooks/pre-push
# Validates branch names
# Compiles Scala code
# WYX-CLI adds additional validation layer
```

**Integration Strategy**:
- WYX-CLI validates BEFORE git operations
- Git hooks validate DURING git operations
- Two layers of validation ensure compliance
- WYX-CLI can bypass checks if needed (with warnings)
